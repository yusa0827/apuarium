# Git リモートの設定とプッシュ確認手順

このプロジェクトでは既定で `origin` などのリモートは設定されていません。以下の手順でリモートを追加し、プッシュできているか確認してください。

## 1. リモートの追加
```
git remote add origin <リモートリポジトリのURL>
```
設定されているリモートは次のコマンドで確認できます。
```
git remote -v
```
URL が表示されない場合は、まだリモートが登録されていない状態です。

## 2. 初回プッシュ
作業中のブランチをリモートへ初めて送る場合は、追跡関係を作成するため `-u` オプションを付けます。
```
git push -u origin <ブランチ名>
```
これにより以後は `git push` だけで同じリモートへプッシュできます。

## 3. プッシュ結果の確認
プッシュ後に次のコマンドでリモートの状態を確認します。
```
git status
```
`Your branch is up to date with 'origin/<ブランチ名>'` と表示されればプッシュが成功しています。また、Git ホスティングサービス上でコミットが反映されているかをブラウザで確認するのも確実です。

## 4. よくあるつまずき
- **リモートが存在しない/URL が間違っている**: `git remote -v` で URL を再確認し、必要に応じて `git remote set-url` で修正します。
- **認証エラー**: SSH 鍵やアクセストークンなど、ホスティングサービスの認証設定を見直してください。
- **ブランチ名のタイプミス**: ローカルのブランチ名とプッシュ先のブランチ名が一致しているか確認してください。

以上の手順で、ローカルからリモートへのプッシュ状況を確実に把握できます。
